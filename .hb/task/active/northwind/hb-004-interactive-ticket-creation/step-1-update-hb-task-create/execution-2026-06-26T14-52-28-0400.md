## Outcome

Before this step, `/hb-task-create northwind/hb-foo` with no flags silently created an empty skeleton and told the user to write a `ticket.md` themselves. After this step, the same invocation enters interactive mode: the skill follows the shared `interactive-ticket-subflow.md`, elicits ticket content from the user, writes it to `/tmp/hb-ticket.md`, and passes it to the SDK via `--ticket` — seeding the skeleton with a real ticket.

Single-file change to `skills/hb-task-create.md` (+35 lines, −8 lines). Additive for `--no-interactive`; behavior-preserving for `--ticket` and `--help`. No new runtime dependency; manifests unchanged.

**Unchanged within `hb-task-create.md`:** Step 1 (help check), the `hb-sdk task create` command itself, `--ticket-overwrite` forwarding logic, the `## Output` section, and the `!cat references-toc.md` expansion line.

**Design as built:** Three-tier flag-precedence block at Step 2. Tier 1 (`$TICKET_SUPPLIED`) exits to SDK with user path; tier 2 (`$NO_INTERACTIVE`) exits to SDK skeleton-only; tier 3 (default) invokes the subflow, captures `/tmp/hb-ticket.md` as `$WRITTEN_TICKET`, and exits to SDK with that path. The subflow is invoked by reference (Claude reads and follows it at runtime); no import mechanism or shell call is needed. The `allowed-tools` frontmatter now includes both `/tmp/*` and `/private/tmp/*` to cover macOS symlink resolution.

No new dependency. Lockfile unchanged.
