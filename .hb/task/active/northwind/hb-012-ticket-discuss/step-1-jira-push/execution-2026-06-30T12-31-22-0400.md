## Outcome

The **motivating case** is now resolved: a user who has just drafted a standalone ticket can have it *become* a Jira issue without leaving the flow. Before, `/hb-ticket-discuss` only printed the ticket to stdout for manual copy-paste into Jira's UI; after, the skill offers to push, and on the **primary path** creates or updates a real Jira issue via the connected Atlassian MCP — using the generated ticket as the issue description (Markdown) — and reports the issue key and browse URL.

The **general result**: one new interactive branch with three-tier precedence (push → no-push fallback → MCP-unavailable fallback). The scope boundary held — **one file changed** (`skills/hb-ticket-discuss.md`); the shared `interactive-ticket-subflow.md` and its two sibling callers (`hb-task-create`, `hb-task-step-add`) were untouched, and **no raw Jira REST API** was introduced. The change is **additive**: the pre-existing stdout emit is preserved verbatim and re-homed under the no-push branch. No automated build/test harness exists in this markdown-procedure repo; structural verification is authoritative and passed.

**Design as built.** Step 3 detects a connected Jira create-issue tool (by capability; `createJiraIssue` is the verified reference in this environment) and, if present, offers create / update / no. Step 4 (primary) resolves `cloudId` (auto-pick if single site, else prompt), then for **create** resolves `projectKey`/`issueTypeName` (default `Task`)/`summary` (proposed-and-confirmed, never silently guessed) and calls `createJiraIssue` with the ticket as a Markdown `description`; for **update** prompts for `issueIdOrKey` and calls `editJiraIssue` setting `fields.description`. On success it sets `$JIRA=pushed` and reports key/URL; on any tool failure it surfaces the error and falls through to Step 5. Step 5 is the reused step-0 stdout emit, gated to `unavailable`/`declined`/post-failure and skipped on a successful push (with an added "MCP unavailable" note when `unavailable`). Step 6 branches the closing message on `$JIRA=pushed`.
