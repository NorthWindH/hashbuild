## Outcome

Before this step, `hb-sdk task unarchive` did not exist; running it produced `error: argument <action>: invalid choice: 'unarchive'`. After this step, `hb-sdk task unarchive <name>` moves a task from `task/archive/<author>/` to `task/active/<author>/`, removes an empty archive author directory, and prints `report_paths` output — completing the archive/unarchive round-trip. Change is additive only: no existing command behavior, signatures, or callers changed. Test count went from 81 → 91 (10 new unarchive tests); all 91 pass.

**Design as built:** `cmd_task_unarchive` is the exact structural mirror of `cmd_task_archive`. It uses `_find_matching_task_folders` (which searches both `active/` and `archive/` subtrees), asserts the found path is under `archive/`, creates `active/<author>/` with `mkdir(parents=True, exist_ok=True)`, asserts the destination doesn't exist, renames, removes the archive author dir if now empty, then calls `report_paths`.
