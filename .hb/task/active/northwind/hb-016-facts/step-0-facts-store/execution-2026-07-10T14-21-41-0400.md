**Design as built:** exactly as planned — `write_facts` calls

with no `.hb/` assertion, returning `""` if either the file or `.hb/` itself
is absent. `cmd_facts_read` uses `print(read_facts(), end="")` to avoid
injecting a newline that would break exact round-trip.

## Deviations from plan

None of substance. One in-flight fix to the plan's own test-spec text: the
plan's `test_facts_write_requires_content_arg` example implicitly called
`facts_write(tmp_path, "", ok=False)`, but an empty string is a valid
positional argument to argparse (it doesn't trigger the "required positional
missing" error path the test is meant to check). Implemented instead as a
direct `run(["facts", "write"], tmp_path, ok=False)` call with no content
argument at all, which is what actually exercises argparse's required-arg
enforcement. The assertion (`returncode != 0`) and AC coverage are unchanged.
