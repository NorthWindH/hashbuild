## Outcome

`hb-sdk idea ...` previously returned an unrecognized-command error; it now executes `add`, `remove`, `show`, and `set-content` correctly. A caller can round-trip an idea through `add → show → set-content → remove` with expected JSON at each stage. Storage is per-author at `.hb/idea/<author>/ideas.json`; the directory is created on first write. Implementation is additive only — no existing subcommand or behavior changed. All 166 tests pass (19 new, 147 existing unchanged).

**Design as built:** `idea.py` follows `task.py` pattern exactly. `path_idea_dir`/`path_idea_file` call `path_hb_asserted()` for `.hb/` guard. `_load_idea_file` returns `{"ideas": []}` when file absent (not an error). `_save_idea_file` creates the author dir with `mkdir(parents=True, exist_ok=True)`. `cmd_idea_show` calls `path_hb_asserted()` explicitly for the no-target branch so the `.hb/` guard fires even when there are no files to load. Index is derived from array position at output time — never stored on disk.

---

## Deviations from plan

None of substance. The implementation followed the plan exactly. One minor in-flight refinement: `cmd_idea_show` calls `path_hb_asserted()` at the top of the function unconditionally (not just in the `target is None` branch). This ensures the `.hb/` guard fires for all three dispatch paths, making the no-hb test reliable for all show variants. The plan specified calling it in the None branch; putting it first is strictly more correct and doesn't change any other behavior.
