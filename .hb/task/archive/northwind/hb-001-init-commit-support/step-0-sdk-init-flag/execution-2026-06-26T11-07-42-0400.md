## Outcome

Before this step, `hb-sdk commit write-message-file` required `--task`, making it impossible to generate a commit message with no task reference (the case `hb-init` needs). After this step, the command has three subparser modes — `plain`, `task`, `task-step` — where argparse enforces per-mode required arguments and rejects flags that don't belong to that mode. `hb-sdk commit write-message-file plain --short "initialize hashbuild"` now exits 0 and produces `hb: initialize hashbuild\n`. The `task` and `task-step` modes produce byte-identical output to the old flat-flag invocations. Test count: 88 → 100 (12 added: 3 updated existing, 9 new). Additive-only change to the CLI interface: no pre-existing behavior changed for valid invocations.

**Design as built:** `_def_cli_commit` adds a second subparser level (`wmf_subs`) beneath the existing `commit_subs`. Each mode parser (`p_plain`, `p_task`, `p_task_step`) defines only the flags it owns — argparse rejects unlisted flags as `unrecognized arguments` without any application-level guard needed. `_commit_write_message_file(subject, long)` is the single message-file write path shared by all three modes. Dispatch flows via `set_defaults(func=...)` through the existing `main()` `args.func(args)` call — no changes to `main()`.
