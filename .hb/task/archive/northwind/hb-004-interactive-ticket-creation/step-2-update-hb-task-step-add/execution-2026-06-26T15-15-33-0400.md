## Outcome

Before this step, `/hb-task-step-add` had no interactive ticket creation: invocations without `--ticket` called the SDK immediately and produced a skeleton-only step with no ticket content, requiring the user to manually fill in `ticket.md`. This step mirrored the three-tier flag-precedence design from `hb-task-create` (shipped in step 1) into `hb-task-step-add`, making interactive mode consistent across both creation entry points. Additive only ‚Äî no pre-existing `--ticket` or `--flavor` behavior was changed. Single file edited; no SDK, no other skill, no other file changes.

**Design as built:** Three-tier precedence in step 2: `$TICKET_SUPPLIED` (check first) ‚Üí `$NO_INTERACTIVE` ‚Üí interactive mode. The `$TICKET_SUPPLIED` guard fires before any interactive logic, preserving the existing `--ticket` path unconditionally.
