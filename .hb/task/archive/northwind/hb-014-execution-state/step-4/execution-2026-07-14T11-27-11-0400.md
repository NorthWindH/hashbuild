## Outcome

`./install` now patches a `/hb-flow` SessionStart hook into the Claude harness's
`settings.json` alongside its existing read-permission patching. Before: a new
Claude Code session or `/clear` in a hashbuild-initialized repo gave no nudge
toward `/hb-flow`. After: the session start prints
`hashbuild: run /hb-flow to see where things left off.` whenever `.hb/` exists
in the session's cwd, and stays silent everywhere else — verified via a
scratch-`HOME` smoke test. Purely additive: no existing `install` behavior
(skill symlinking, permission patching) changed shape. `uv run ruff check`,
`uv run ruff format --check`, and the full `uv run pytest` suite (215 tests)
are clean.

## Deviations from plan

None of substance. Implementation matches the plan's `HookPatcher`
specification (§2), wiring (§3), and test list (§5) as written. One
in-flight adjustment: the plan's non-disturbance test (§5) asserted the
*entire* settings dict is byte-for-byte unchanged after an uninstall-hook
cycle; in practice `SettingsPatcher`'s pre-existing permission-uninstall path
leaves behind an empty `{"permissions": {"allow": []}}` key even when nothing
was removed (this is prior, untouched behavior — `SettingsPatcher._update`
always `setdefault`s `permissions.allow` before diffing). The test was scoped
to compare only the `"hooks"` subtree, which is what this step's non-disturbance
claim is actually about; the empty-permissions-key quirk is pre-existing and
out of this step's scope (noted under Honest notes below).

## Tests added

- `test_fresh_install_adds_hook` — happy path: fresh install creates the matcher group with our entry.
- `test_install_is_idempotent` — the motivating case (AC2): second run reports "already present", no duplicate group/entry.
- `test_install_appends_into_existing_matcher_group` — precedence: an existing `startup|clear` group with an unrelated hook gets our entry appended, not a new group.
- `test_uninstall_removes_hook_cleanly` — AC3: after install+uninstall, `hooks` key is fully pruned.
- `test_uninstall_does_not_disturb_unrelated_hooks` — AC3: unrelated `PreToolUse` hook and an unrelated `SessionStart` (`"startup"`-only) group are byte-for-byte unchanged across an install+uninstall cycle.
- `test_install_confirm_prompt_shows_hook_diff` — AC4: stdout shows the diff line naming the matcher and exact command before the write.
- `test_hook_command_silent_outside_hb_dir` — negative case: the hook command itself is silent (exit 0, empty stdout) outside `.hb/`.
- `test_hook_command_prints_reminder_inside_hb_dir` — positive case: the hook command prints the exact reminder message inside `.hb/`.

Pre-existing `tests/skills/scripts/hb_sdk/` suite (207 tests) passed unchanged, confirming this step never touched `hb-sdk`.
