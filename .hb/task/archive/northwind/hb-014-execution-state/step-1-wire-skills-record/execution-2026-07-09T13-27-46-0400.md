## Outcome

`hb-sdk state record`/`state show` (shipped in step-0) is now wired into all 8
`hb-*` skills that mutate task/step state; `.hb/.state.ignore.json` is written
after every successful run of these skills and after the 3 non-exempt
documented failure stops, so `hb-sdk state show` now reflects real progress
instead of always reporting "No recorded state." The motivating design
change also landed: `hb-sdk state record` now self-generates its timestamp
(`datetime.now().astimezone().isoformat()` — local time, timezone-aware) and
the `--timestamp` CLI flag is removed entirely, so no skill needed a new
`Bash(date *)` permission. Scope boundary held: all 8 skill edits are
additive (new steps/bullets only, existing prose/commit mechanics/error text
byte-identical); the only breaking change is the `--timestamp` CLI removal,
contained to `state.py` and its own test file. Full test suite green (182
passed), including the updated `test_hb_sdk_state.py`.
