## Outcome

`hb-sdk summarize` previously emitted JSON only; any consumer wanting the rendered status view had to re-implement the rendering logic. This step moves all rendering into `hb-sdk summarize --format md`, making `hb-status` a thin passthrough and retiring `status-template.md`. The JSON output path is untouched — additive only. 31 existing tests plus 19 new tests green; scope limited to 6 intended files.

**Design as built:** `_build_data(hb)` contains all data-gathering logic (previously inlined in `cmd_summarize`). `cmd_summarize` is now 4 lines: call `_build_data`, branch on `args.format`. `_render_md(data)` builds a `lines` list, appends each section conditionally, then joins. `_next_action(data)` implements the 8-step decision tree from the former `status-template.md` comment.

---

## Deviations from plan

The plan specified refactoring by moving the existing `cmd_summarize` body into an `else:` branch. Instead, the data-gathering logic was extracted into a dedicated `_build_data(hb: Path) -> dict` function. This achieves the same clean separation between data gathering and output formatting but is slightly cleaner (avoids deeply nested `else:` block). The JSON output is byte-for-byte identical; no behavioral change.

Plan estimated ~80 lines of new rendering logic; actual was ~130 lines, because `_next_action` required 8 explicit conditions and `_render_md` included all sections with their conditional guards.
