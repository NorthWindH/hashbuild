## Outcome

`hb-task-step-review-address.md`'s per-item addressing loop (section 9) now reads and reconciles the facts store exactly once per review item, matching the pattern already shipped for planning (step-1) and execution (step-2). Before this change, addressing a review item never saw `.hb/facts.md` and could never update it — any fact rendered stale while resolving a concern would silently rot. After this change, section 9 reads `$FACTS` at the start of each item (new 9a), instructs the model to apply it while addressing the concern (9c), and reconciles the store — pruning stale facts, adding durable new ones — before that item's own commit (new 9f, before 9g). Scope boundary held: purely additive to section 9 of one file — no other section, no Python module, no test file touched. Static verification (grep counts, heading sequence, `git diff --stat`) all match the plan's expected output exactly.
