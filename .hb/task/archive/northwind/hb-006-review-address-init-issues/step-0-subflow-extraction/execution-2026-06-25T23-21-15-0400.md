## Outcome

The `review-init` subflow was extracted into a new passive shared document `skills/references/review-init-subflow.md`. Before this change, `review-address` invoked `review-init` as a full skill when `review.md` was missing, causing the agent to receive a "fill in review items and re-run" notification and stop prematurely (Bug 1), and an intermediate commit from `review-init` shifted `HEAD` before the TODO REVIEW scan in step 4 (Bug 2). After this change, `review-address` step 3 inlines the file-creation subflow via `!` injection with no notification and no commit, so the agent continues directly to step 4 with HEAD pointing at the user's work commit. The change is additive only — standalone `review-init` behaviour is identical to before (same resolve → check → create → notify → commit sequence).
